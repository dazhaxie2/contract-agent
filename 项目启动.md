# 项目启动方式说明

根据项目的架构设计，启动该项目主要有两种方式。由于本项目依赖较多（数据库、向量库、消息队列等），**强烈推荐使用 Docker 方式一键启动**。

## 方法一：Docker 方式（推荐，一键启动）

这是最简单的方法，会根据 `docker-compose.yml` 自动启动后端、前端以及所有依赖服务（PostgreSQL, Redis, Milvus, MinIO, Kafka 等）。

1. 确保已安装 Docker 和 Docker Compose
2. 在项目根目录执行启动命令：

```
docker-compose up -d
```

### 启动后访问地址

- 前端地址: http://localhost:3000

- 后端 API: http://localhost:8000

- 监控台 (Grafana): 

  http://localhost:3001

  - 初始账号：`admin`
  - 初始密码：`admin123`

  

## 方法二：本地开发方式（手动启动）

如需在本地直接运行代码进行调试，按以下顺序操作：

### 1. 准备基础服务

仅启动 Docker 中的依赖项：

```
docker-compose up -d hidb redis milvus minio kafka jaeger prometheus
```

### 2. 启动后端（Python/FastAPI）

1. 进入 `backend` 目录
2. 创建并激活虚拟环境，安装依赖：

```
pip install -r requirements.txt
```

1. 配置环境变量：参考 `app/core/config.py` 默认值或新建 `.env` 文件
2. 运行后端服务：

```
uvicorn app.main:app --reload --port 8000
```

### 3. 启动前端（React/Vite）

1. 进入 `frontend` 目录
2. 安装依赖：

```
npm install
```

1. 启动开发服务器：

```
npm run dev
```

前端运行地址：http://localhost:3000

## 关键路径提示

- 后端入口: `backend/app/main.py`
- 前端入口: `frontend/src/main.tsx`
- 配置文件: `backend/app/core/config.py`、`frontend/vite.config.ts`

## 环境要求注意

本地启动前，请确保系统已配置：

- Python 3.9+
- Node.js 18+
