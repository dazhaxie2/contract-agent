# AI 智能体开发笔记：48 天构建日程与费用管理工具

## 开发背景与动机  

- 团队因缺乏秘书导致日程管理混乱，成员对工作安排认知不一致，无法明确出差、录片、参会等时间节点。

- 现有日程管理软件存在操作繁琐问题：创建 / 修改日程需填写标题、时间、地点等表单，流程冗长易遗忘初衷。

## 产品构想与核心功能  

- **交互方式**：模拟微信对话框，支持自然语言输入需求，AI 自动处理并返回结构化结果。

- **核心功能**：

  - 日程管理：如创建 "4 月 18 日 AI 行业研讨会"、查询北京车展时间并关联出差行程、清空下周一行程并请假、自动安排团队会议（含时间冲突处理）。

  - 费用管理：关联出差代驾发票并报销、记录团队咖啡费用（150 元）。

  - 跨域协同：实现日程与费用的有机结合，如将代驾发票与上海出差事件关联。

## 现有工具局限性分析  

- **大模型**：存在记忆窗口限制，无法处理超出窗口的日程，高频调整后难以精准定位和修改，缺乏结构化数据。

- **Siri**：仅支持单一增删改查操作，无法处理多意图、批量操作及结合互联网搜索，无费用管理功能。

- **传统工具**：基于表单交互，需手动选择类别（如娱乐餐饮、日常生活费用），操作流程冗长；多为 "AI 套壳" 产品，聊天机器人与数据库隔离，非原生 AI 架构。

- **其他工具**：需妥协隐私性（开放大量权限）和经济性（本地存储消耗 token），不具备市场投放条件。

## 产品架构设计  

- **三大组成部分**：

  - 大模型：作为 "脑子" 负责思考，处理意图解析与推理。

  - Agent：负责本地数据库及工具调用，循环推理并完成任务。

  - 图形界面：解决大模型延迟问题，提供快速查询入口（如滚动菜单式日程列表、月视图切换），仅负责数据呈现，不处理输入。

## 技术实现原则  

- **薄客户端原则**：iOS 端仅负责前端渲染，所有业务逻辑（增删改查、Agent 循环、工具调用）在云端处理，便于跨平台拓展（安卓、桌面端）。

- **混合界面策略**：聊天界面为 iOS 原生（滑动组件、输入框等），日历视图使用 Web 页面，通过桥接与原生硬件能力结合，接近原生体验。

- **领域驱动设计（DDD）**：日程管理与费用管理为独立业务域，由顶层编排器 Agent 协调跨域查询与任务下发，实现高内聚低耦合。

- **计划与执行流水线**：将用户需求（批量 / 单一操作）转化为计划，大模型统一推理解析后，用户确认即可执行数据库操作，提升效率（如批量创建事件从 1.5-2 分钟缩短至毫秒级）。

- **Smart Agent, Dumb Tools**：Agent 掌握上下文并推理决策，工具仅负责简单执行（如创建、删除、更改），剥离校验逻辑，降低工具复杂度。

## 可观测性与工程品质  

- **决策 ID 追踪系统**：通过日志记录 Agent 行为，便于快速定位问题（如 "击鼓传花" 中锁定掉花的 "小朋友"），辅助大模型排查 bug。

- **自改进飞轮**：用户报错自动生成测试用例，在本地环境复现，通过 WebSocket 投送至 iOS 模拟器执行回归测试（100 + 用例），确保数据结构、效应及 UI 渲染正确。

## 设计体系与技术选型  

- **令牌化设计（Tokenized Design System）**：将字体、字号、颜色、间距等转化为设计令牌（Design Token），通过 JSON 加载到组件，实现界面规范统一与快速生成。

- **工具链**：使用 Pencil MCP + Cloud + Mobile 获取设计灵感，解决 CLI 界面无法实时预览问题。

- **技术栈选择**：

  - Agent 框架与 SDK：采用 Versus SDK，处理工具循环、上下文及结构化输出。

  - 数据库：为规模化与多租户拓展预留架构，确保数据安全稳定。

  - 前端：部分原生（聊天界面）+ 部分 Web（日历视图），平衡开发效率与体验。

  - API：Anthropic 全家桶（Sonnet 模型）、高德地图 API（地址补全与搜索）、千问 Embedding V4（向量匹配）、General Reinker（评分仲裁）。

## 开发成果与反思  

- **成果**：48 天开发 12 万行有效代码，实现日均使用的日程与费用管理工具，支持自然语言交互、跨域协同，开发效率较传统 iOS 模式提升 5-6 倍。

- **反思**：

  - 非技术背景开发者可借助 AI 工具快速弥合与专业程序员的差距，创造力、用户体验理解及市场洞察成为核心竞争力。

  - 新时代工具（大模型、AI 编程）需结合新思维，提出更优解决方案，而非填补需求空白。
