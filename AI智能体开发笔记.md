# AI Agent 系统通用开发指南

> 从0构建一个生产级智能体系统的架构方法论与实践清单，适用于任何领域的 Agent 项目。

---

## 一、产品定义：对话优先，表单辅助

### 核心交互模式
- **主入口**：自然语言对话（模拟 IM 交互），用户无需理解系统内部结构。
- **辅助入口**：结构化页面（列表、流程、仪表盘等）负责数据呈现和快速查询，不负责输入。
- **原则**：用户只需"说出需求"，系统负责意图理解、任务拆解和执行。

### 产品架构三件套
1. **大模型（Brain）**：意图解析、推理、生成，是系统唯一的"思考层"。
2. **Agent（Hands）**：工具调用循环、上下文管理、任务编排，连接大脑与数据。
3. **图形界面（Eyes）**：纯呈现层，解决大模型延迟问题，提供即时的可视化入口。

## 二、Agent 架构设计

### 2.1 编排器 + 子Agent 协同

```
用户请求 → OrchestratorAgent（意图分析 + 路由）
                ├→ RetrievalAgent（检索）
                ├→ ComplianceAgent（合规分析）
                ├→ ComparisonAgent（比对）
                ├→ DraftingAgent（起草）
                ├→ ValidationAgent（校验）
                └→ 直接处理（简单查询）
```

**设计原则**：
- **Orchestrator 不做具体业务**：只做意图识别、任务拆解和结果综合。
- **子 Agent 专注单一领域**：每个子 Agent 有独立的 system prompt、工具集、可选模型。
- **工具保持"笨"**：工具只做简单执行（CRUD、检索、计算），不含校验逻辑。校验在 Agent 层。

### 2.2 ReAct 循环

```
Thought → Action（调用工具）→ Observation → Thought → ... → Final Answer
```

**关键参数**：
| 参数 | 推荐值 | 说明 |
|------|--------|------|
| max_iterations | 10 | 防止死循环 |
| max_execution_time | 300s | 全局超时 |
| temperature | 0.1-0.3 | 生成稳定性 |

### 2.3 计划确认机制

对于批量操作或不可逆操作：
1. Agent 先生成执行计划（Plan），列出每步操作。
2. 用户确认后才执行数据库写入。
3. 将批量操作从分钟级压缩到毫秒级。

## 三、RAG 检索增强架构

### 3.1 分层 RAG 架构选型

| 层级 | 技术 | 解决的问题 |
|------|------|-----------|
| 基础检索 | Multi-Index Hybrid RAG | 向量 + 关键词 + 图谱三路并行 |
| 纠错机制 | Self-RAG + CRAG | 模型自判断是否需要检索 + 召回校验 |
| 深度推理 | Graph RAG | 实体关系推理，解决逻辑关联 |
| 任务调度 | ReAct 多智能体 | 复杂任务拆解与编排 |

### 3.2 文档预处理与分块

```
文档上传 → 格式解析 → 结构识别 → 清洗去噪 → 元数据提取 → 混合分块 → 增强处理
```

**分块策略选择**：
| 文档类型 | 策略 | 原子单位 |
|---------|------|---------|
| 法律/标准文档 | 结构分块 | 条/款 |
| 合同 | 合同分块 | 章/条 |
| 无结构文本 | 语义分块 | 128-1024 token |
| 超长内容 | 摘要分块 | 原文 + 摘要双向量 |

**关键**：每个块附加元数据、层级路径、实体标签，构建父子块关系用于上下文回溯。

### 3.3 多阶段检索流水线

```
Query预处理（意图识别+改写+实体抽取）
    ↓
三路并行检索（向量Top50 + 关键词Top30 + 图谱Top20）
    ↓
RRF融合去重 → 粗排（轻量模型→Top30）→ 精排（重排模型→Top10）
    ↓
CRAG校验（过滤失效内容）+ Self-RAG（不足则二次检索）
    ↓
上下文窗口构建（层级回溯 + 优先级排序 + Token管控）
```

## 四、记忆与上下文管理

### 4.1 三层记忆体系

| 层级 | 类型 | 生命周期 | 存储 |
|------|------|---------|------|
| 短期 | 对话历史 | 滑动窗口（5-10轮） | 数据库 |
| 中期 | 会话摘要 | 超量时压缩 | 数据库 |
| 长期 | 用户画像 + 领域知识 | 跨会话持久 | 数据库 + RAG |

### 4.2 事实抽取

- **初级**：正则模式匹配（`key: value`、`X是Y`）。
- **进阶**：LLM 结构化抽取（用轻量模型提取 `key | value` 对，更准确）。
- **原则**：抽取后去重、带置信度、可跨会话复用。

### 4.3 上下文窗口分层管控

| 区段 | Token 分配 | 内容 |
|------|-----------|------|
| System Prompt | 2K | 角色定义、规则约束 |
| Memory | 4K | 对话历史 + 摘要 + 事实 |
| Retrieval Context | 16K | 检索结果 + 引用 |
| Generation | 8K+ | 生成预留 |

## 五、零幻觉保障体系

### 5.1 提示词三层架构

1. **System Prompt（固定）**：角色 + 硬性约束（"100%基于检索上下文，禁止编造"）。
2. **Task Prompt（任务专项）**：不同任务类型的输出格式与校验规则。
3. **Dynamic Prompt（实时注入）**：检索上下文 + 对话记忆 + 用户画像。

### 5.2 生成-校验闭环

```
LLM生成 → 事实性校验（小模型判断引用准确性）
    ├→ PASS → 返回用户
    ├→ PARTIAL → 标记不确定部分
    └→ FAIL → 触发修正或补充检索 → 重新生成
```

### 5.3 LLM-as-Judge 评分

每次执行后，用轻量模型对输出评分：
- **Relevance**（相关性 0-1）
- **Factuality**（事实性 0-1）
- **Completeness**（完整性 0-1）
- **Clarity**（清晰度 0-1）

评分写入数据库，支撑持续优化闭环。

## 六、可观测性与工程品质

### 6.1 决策追踪

- 每个请求分配唯一 `trace_id`，覆盖全流程（检索→Agent→生成→返回）。
- 每个步骤记录 `span`：耗时、输入输出、Token 消耗、异常。
- 技术选型：OpenTelemetry + Jaeger/Prometheus + Grafana。

### 6.2 执行持久化

| 表 | 用途 |
|----|------|
| agent_executions | 每次执行的完整记录 |
| agent_steps | ReAct 每步详情 |
| retrieval_logs | 检索三路调试数据 |
| citation_records | 引用溯源链 |

### 6.3 自改进飞轮

```
用户报错 → 自动生成测试用例 → 本地复现 → 回归测试 → 确认修复
```

## 八、高并发与高可用

### 关键策略

1. **无状态微服务**：所有服务可水平扩展，状态存分布式存储。
2. **读写分离**：数据库、向量库、图谱库均读写分离。
3. **三级缓存**：本地内存 → Redis → 向量库冷热分离。
4. **异步化**：入库、日志、记忆更新走消息队列。
5. **限流熔断**：Sentinel 限流 + Circuit Breaker 熔断降级。
6. **灰度发布**：新版本先小流量验证，指标正常再全量。

## 九、评估体系

### 离线评估指标

| 指标 | 达标要求 |
|------|---------|
| Top10 召回率 | ≥ 95% |
| Top10 精确率 | ≥ 85% |
| MRR | ≥ 0.9 |
| NDCG@10 | ≥ 0.9 |

### 在线评估指标

| 指标 | 达标要求 |
|------|---------|
| 用户采纳率 | ≥ 80% |
| 负反馈率 | ≤ 1% |
| 专家评分 | ≥ 8/10 |

### 优化闭环

```
自动化评估流水线（CI/CD触发）
    → 离线评估报告
    → 负反馈自动入测试集
    → A/B 测试新策略
    → 模型定期微调
```
